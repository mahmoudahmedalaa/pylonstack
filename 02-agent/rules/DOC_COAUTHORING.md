# Documentation Co-authoring Standards

When documenting system architecture or user-facing guides:
- **Audience First:** Determine if the doc is for users, developers, or architecture.
- **Top-Down Structure:** Start summarizing WHAT it is, then WHY it is needed, then HOW it works.
- **Code is not Doc:** Do not just duplicate the code into comments. Explain the architectural decisions.
- **Living Docs:** During every phase of development, documentation MUST be manually kept up to date alongside code.
- **Automated Validation:** If documentation references code types, ensure it aligns with our TS definitions.
